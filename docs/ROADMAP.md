# Megido Roadmap

This roadmap is community-driven and will evolve as researchers contribute findings, fixes, and new ideas.

## Short-Term Goals (0–3 months)

- Improve onboarding and contributor documentation
- Stabilize core scanner workflows and reduce high-noise findings
- Expand issue templates and triage process for faster feedback loops
- Improve reproducible bug reports and evidence collection paths

## Mid-Term Goals (3–9 months)

- Add new scanner modules for common web/API/mobile vulnerability classes
- Improve authentication and session testing depth
- Expand reporting/export options (JSON/HTML/other formats)
- Strengthen UI usability for day-to-day research workflows

## Long-Term Goals (9+ months)

- Mature plugin/module ecosystem for community extensions
- Improve mobile app testing and hybrid app coverage
- Add deeper integrations with external security workflows/platforms
- Improve scalability and team-friendly deployment patterns

## Open Areas for Contributors

We actively welcome contributions in:

- New scanner modules and verification logic
- Mobile-app testing improvements
- Plugin system design and extension points
- Reporting/export format enhancements
- UI/UX polish and accessibility improvements
- Documentation, examples, and contributor onboarding
- Integrations with developer/security tooling

## How to Contribute to the Roadmap

- Open issues for concrete improvements.
- Propose larger changes via Discussions or RFC-style issues.
- Submit PRs for roadmap items you want to own.
