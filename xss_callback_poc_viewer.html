What is Callback-Based Verification?

Instead of relying on alert() dialogs or console errors, our system injects payloads that make HTTP callbacks. Only when we receive the callback do we mark the XSS as VERIFIED. This provides concrete proof of exploitation suitable for bug bounty submissions.

Benefits of Callback Verification

  • Reduced False Positives: Only report XSS that actually executes
  • Proof of Exploitation: HTTP callback provides concrete evidence
  • Bug Bounty Ready: Reports include all necessary verification details
  • Professional Reports: Timestamps, IPs, and interaction logs included
