## Reporting a Vulnerability (Private)

If you discover a vulnerability in **Megido itself**, please report it privately first:

1. **GitHub Security Advisories** (preferred)
   - Use the repository Security tab and submit a private report.
2. **Email fallback**
   - `security@example.com` (**placeholder** — maintainers should replace this with a real contact).

Please include:

- Affected version/commit
- Impact and severity assessment
- Reproduction steps or proof of concept
- Suggested mitigation (if available)

## Expected Response Times

- Initial acknowledgement: within **48 hours**
- Follow-up status update: within **7 days**
- Target fix timelines:
  - Critical: 7–14 days
  - High: 14–30 days
  - Medium: 30–60 days
  - Low: 60–90 days

## Responsible Disclosure

- Report vulnerabilities privately first.
- Allow maintainers reasonable time to investigate and patch.
- Public disclosure should occur after a fix or coordinated timeline agreement.
- Researchers will be credited unless anonymity is requested.

## Safe Harbor for Good-Faith Research

We support good-faith security research intended to improve Megido and user safety. If you:

- Avoid privacy violations, data destruction, and service disruption
- Test only within authorized scope
- Report findings promptly and responsibly

…we will treat your research as authorized under this policy and will not pursue action for accidental, good-faith outcomes that are promptly disclosed.

## Scope Clarification

This policy is for vulnerabilities in Megido code and infrastructure.

It does **not** cover intended security-testing capabilities (for example, request crafting, scanning, interception, or replay features used in authorized assessments).

## Legal / Ethical Use Reminder

Megido must only be used against systems you own or systems where you have explicit permission to test.
